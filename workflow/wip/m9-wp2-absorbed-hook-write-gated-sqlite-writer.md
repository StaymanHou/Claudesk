## Problem Statement

M9 absorbs the standalone `claude-time` tracker into Claudesk. WP1 (probe) froze the segment-model contract and enumerated the exact hook delta. **WP2 builds the substrate the reclassifier (WP3) will read: extend Claudesk's CC hook from 4→10 events, add 4 wire fields, and persist every event into a per-identity SQLite DB — gated so it only writes when tracking is ON.** The write path must be a *second, gated* consumer of the existing `HookEvent` stream: the M3 status machine (idle/running/awaiting dots) MUST stay byte-for-byte unchanged. Per the absorption decisions, the Perl hook stays a pure socket-forwarder (no `sqlite3` subprocess like claude-time's) — the **Rust `time_store` module owns the DB write**. The `events` schema must carry a `source` discriminator (`cc-hook` vs `claudesk-native`) so WP2.5 can write native-signal rows into the same store. Privacy invariant throughout: prompt **length only, never text**; never tool I/O.

**Inputs (frozen by WP1):**
- `docs/product/wp1-time-analytics-probe-outcome.md` §(d) — the 4→10 event delta + 4 wire fields + `PostToolUseFailure` confirmed distinct + the SQLite `events(ts, session_id, cwd, event, tool_name, agent_type, meta)` schema.
- `docs/product/wbs.md` → WP2 (tasks) + WP2.5/WP5 notes (schema must be WP2.5-source-aware; the write-gate is read by WP5's toggle later).
- Reference: `_ref/claude-customization/tools/claude-time/hook.pl` (the per-event field extraction + `meta` JSON shape + WAL/`.timeout` bootstrap) — Claudesk ports the *field mapping* and *schema*, NOT the direct-sqlite3 write (Rust owns the write).

**Key seam facts (surveyed):**
- `hook_install/mod.rs::CLAUDESK_EVENTS` = 4 events; registration is additive/idempotent/reversible/per-identity — extend the const, tests re-green.
- `resources/claudesk-hook.pl` forwards a fixed field set + `timestamp`; extend it with per-event field extraction (`prompt_length_chars`, `tool_use_id`, `agent_type` from `subagent_type`, `source`), preserving exit-0-unconditional + privacy.
- `hook_socket/mod.rs::HookEvent` + `parse_line` model the wire; add the new snake_case fields (`#[serde(default)]`) + key-shape tests.
- `status_broadcaster::event_to_state` maps only the 4 current events and drops unmapped ones — a new `PreToolUse`/`SubagentStart`/etc. must NOT flip a dot (verify the drop).
- **The fan-out seam is `lib.rs:194-217`:** `start_on_launch` returns one `mpsc::Receiver<HookEvent>` handed *once* to `start_broadcaster`. `mpsc::Receiver` is single-consumer — adding `time_store` as a second drain requires a tee/fan-out (the accept-loop sends to N senders, or a fan-out thread re-broadcasts).
- New dep: **`rusqlite`** (bundled SQLite) — the one net-new arch element M9 introduces (WBS "Net-new architecture"; scoped exception to "flat JSON, no DB" which governs the *project list*, not this feature-local analytics DB). Reconciled into `arch.md` at WP7.
