## Problem Statement

Claudesk's `feature-verify-self` currently drives **Playwright (MCP) against the bare Vite dev URL**, which has **no Tauri IPC** — so `list_projects`/`cc_spawn`/`listen("workspace-status")` all throw, no `.workspace` ever mounts, and every live-DOM Observable Outcome on a workspace-status surface comes back UNVERIFIED. This has bitten three consecutive workspace-UI features (M4 WP3 filmstrip, WP4 collapse, WP4b focus indicator). M5 (PiP) is *more* live status rendering, so M5 planning is the deliberate moment to decide the testing posture before building another un-agent-verifiable surface. **This is a knowledge-producing probe, not product code:** its deliverable is a written **adopt-or-reject** verdict in `docs/product/wbs.md` "Probe outcomes", backed by a real macOS smoke test of the lead candidate [`hypothesi/mcp-server-tauri`](https://github.com/hypothesi/mcp-server-tauri) (`tauri-plugin-mcp-bridge`, WebSocket :9223, drives the real WKWebView with live IPC). All desk research (candidate, alternatives, hard dev-only-gating requirement, caveats) is already captured in `SURFACE-2026-06-23-VERIFY-SELF-DRIVER-FOR-WORKSPACE-UI` — the probe executes the smoke test and decides. Any plugin wiring added is **dev-only** (`#[cfg(debug_assertions)]` around `tauri_plugin_mcp_bridge::init()`) and may be torn down if the verdict is REJECT.
