## Problem Statement

WP6b-2 lights up the remaining time-shape surfaces â€” **Week**, **Month**, **Custom** (date-range), and the selected-session **SidePanel** â€” as **pure render-surface ports over data WP4 already ships**. There is **NO new backend**: WP4's `time_analytics_query` command already resolves `Day`/`Week`/`Custom` windows (`src-tauri/src/time_store/commands.rs:518` `QueryWindow` enum; `query.rs` `build_week`/`build_range`), returning `WeekPayload` for `week` and `RangePayload` for `day`/`custom`. **Month is a `custom` month-range query** (there is no `month` window kind) that returns a `RangePayload`; the MonthView renders a contribution-grid intensity calendar over it.
