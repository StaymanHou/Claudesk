# Changelog

- **Feature shipped:** WP3a — editor core-editing parity: layered the daily Sublime-parity editing features onto the WP2 editor shell via a new pure `editorExtensions.ts` builder — multi-cursor with Cmd-drag rectangular/column select and Cmd-D select-next, in-file find (Cmd+F) / replace (Cmd+R) via a dark-styled `@codemirror/search` panel, font-size zoom (Cmd+= / Cmd+- / Cmd+0) via a Compartment-swapped `fontSizeTheme` persisted globally in localStorage, a `@replit/codemirror-minimap`, and `scrollPastEnd()`; plus a flex/grid height-chain fix (`.workspace-right` grid-item `min-height:0` + the `@uiw` `.cm-theme` wrapper) so the editor scrolls both axes with word-wrap off.
- **Milestone:** WP3a (Milestone 2, editor core-editing parity) complete — the 3rd of Milestone 2's 12 WPs (after WP1 + WP2); WP3b/3c and WP4–WP10 remain.
- **Feature shipped:** WP3b — editor command palette: a net-new Cmd+Shift+P React overlay over CodeMirror 6's command set whose first command set is syntax/mode selection (Set Syntax: JS/JSX/TS/TSX/Rust/Markdown/Plain Text, overriding the extension-derived language for the open file and resetting on file change), built on an extensible `{id,title,run}` registry composed in EditorPanel and passed into the overlay, opened via the WP1-proven capture-phase document-keydown listener so the chord fires while focus is inside the editor (active-gated to the focused workspace) and a Shift-required predicate that stays distinct from WP6's bare Cmd+P — with a chord-exclusivity matrix and chord-ownership map locking in coexistence for WP5/WP6/WP8.
- **Milestone:** WP3b (Milestone 2, editor command palette) complete — the 4th of Milestone 2's 12 WPs (after WP1 + WP2 + WP3a); WP3c, WP4–WP7, WP10, WP8, WP9 remain.
- **Feature shipped:** WP3c — editor split panes: the right-half editor can now split into N vertically-stacked CodeMirror 6 panes that are viewports onto the _same_ file (a shared-document model — the high-value gesture in a half-width column; independent-file-per-pane was deferred to a follow-up WP), built on a new pure `editorPanes.ts` pane/focus reducer (split / close / focus / collapse, last-pane guard, focus-reassign-on-close) driving a refactored `EditorPanel` that renders N `<CodeMirror>` instances over one panel-level buffer, with a focused-pane tracked via capture-phase focus (chords like Cmd+S and Cmd+Shift+P act on the shared doc), a per-pane close affordance, collapse-back-to-one on opening a different file, and an active-pane accent rendered as a `::before` strip (an inset box-shadow was occluded by CodeMirror's opaque editor — caught and fixed at verify-human).
- **Milestone:** WP3c (Milestone 2, editor split panes) complete — the 5th of Milestone 2's 12 WPs (after WP1 + WP2 + WP3a + WP3b); WP4–WP7, WP10, WP8, WP9 remain.
- **Feature shipped:** WP4 — git diff viewer (Sublime-Merge-style): a view-only diff viewer in the workspace right half (behind the Editor↔Diff toggle) with a collapsible Commits section on top and a scrolling column of collapsible per-file sections showing inline +/− hunks, where a new Rust `git_diff` module (`git2`) computes the changed-file list, per-file diff hunks, a paginated commit history (revwalk, Load-more), and per-commit diffs (vs first parent), and the React frontend renders styled +/− lines directly (no `@codemirror/merge`); built via a full spec→plan→build×2 after the first attempt's file-list+single-diff UX was rejected at verify-human and redesigned to the Sublime-Merge model, operator-approved live in the real app.
- **Milestone:** WP4 (Milestone 2, git diff viewer) complete — the 6th of Milestone 2's 12 WPs (after WP1 + WP2 + WP3a + WP3b + WP3c); WP5–WP7, WP10, WP8, WP9 remain.
